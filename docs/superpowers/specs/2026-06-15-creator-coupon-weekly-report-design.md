## 0. 한 줄 요약

크리에이터 베이직 협찬(쿠폰)이 **실제 콘텐츠 산출로 이어지는지**, 그리고 프로그램이 **건강하게 굴러가는지**를 매주 금요일 11시에 자동 점검하는 HTML 리포트.

---

## 1. 목적 (북극성 + 5대 목적)

> **북극성**: "우리가 제공한 크리에이터 협찬(쿠폰 할인)이 실제 콘텐츠 산출로 이어지는지, 그리고 이 협찬 프로그램이 건강하게 굴러가는지를 매주 점검한다."

| # | 목적 | 주간 액션 |
|---|------|----------|
| ① | **이행 점검** — 퇴실+14일 내 콘텐츠 미제출 크리에이터 색출 | 리마인드 발송 / 협찬 회수 검토 |
| ② | **리스크** — 취소건 모니터링 | 쿠폰 발급 후 취소 = 협찬비 누수·어뷰징 신호 확인 |
| ③ | **활성도** — 쿠폰 사용 수 추이 | 베이직 협찬 프로그램 볼륨 파악 |
| ④ | **협찬 비용 가시화** — 주간 총 할인액 합 | ROI 분모, 예산 추적 |
| ⑤ | **재사용 크리에이터 식별** — 반복 사용 핵심 크리에이터 | 충성 크리에이터 관리, 콘텐츠 검증 우선순위 |
| ⑥ | **캠핑장별 분포** — 협찬 다발 캠핑장 | 캠지기 재참여·만족 신호 |

## 2. 구현 방식

**선택: A안 (에이전트 + MCP)** — v1으로 출시, 필요 시 B안으로 하드닝.

| | A. 에이전트 + MCP (채택) | B. 독립 스크립트 (향후) |
|---|---|---|
| 동작 | 스케줄 에이전트가 `probe`(MongoDB) + `Airtable` MCP로 조회 → HTML 생성 | repo Node 스크립트가 SDK로 직접 연결 |
| 장점 | 신규 시크릿 불필요, **유사매칭에 LLM 판단 활용 가능**, 빠른 출시 | 완전 결정론적, 토큰비 0, cron 안정 |
| 단점 | 매 실행이 LLM, cron에서 MCP 인증 리스크 (검증 필요) | `MONGODB_URI` 시크릿 필요, 코드량↑, 유사매칭 코드 복잡 |

> A안을 택한 결정적 이유: 캠핑장 이름 **유사매칭**("여수 작금캠핑장" ↔ "작금 캠핑장")이 핵심인데, 이건 규칙 코드보다 LLM 판단이 강하다.

## 3. 데이터 파이프라인

```
camping.coupons          (name ~ '크리에이터')         ← 쿠폰 마스터 (쿠폰명, masterCode)
   └─ camping.user_coupons   (isUsed, usedTimestamp)    ← 사용 이력
        └─ camping.bookings  (code, camp/zone, checkIn~checkout, status, couponDiscount, campName)
             └─ Airtable 「인플루언서 컨텐츠 업로드」(tblta2cow9ymKr68J)  ← 유사매칭으로 콘텐츠 유무 판정
```

- MongoDB 접근: `mcp__...cpf__probe_query_run` (camping DB, PAT 인증)
- Airtable 접근: `mcp__Airtable_MCP_Server__list_records` (base `appEGM6qarNr9M7HN`)
- 콘텐츠 테이블 URL: https://airtable.com/appEGM6qarNr9M7HN/tblta2cow9ymKr68J/viwBfxyUvwbGG9IY0

### 3.1 쿠폰→예약 조회 (검증된 aggregate)
`coupons`에서 `name ~ /크리에이터/` → `user_coupons` lookup(`isUsed:true`) → `bookings` lookup. 산출 필드:
`couponName, masterCode, usedTimestamp, discountedAmount, campName, zoneName, bookingCode(code), bookingStatus, checkInTimestamp, checkoutTimestamp, totalCharge, couponDiscount`.

## 4. 두 개의 시간창 (핵심 개념)

하나의 주간 리포트가 **목적별로 다른 창**을 본다. 섞으면 "이번 주 사용"과 "지금 회수할 미제출"이 혼동된다.

| 창 | 범위 | 쓰임 |
|----|------|------|
| **활동창** | `usedTimestamp` ∈ 지난 7일 | ②③④ 활성도·취소·협찬비. "이번 주에 쓰인 쿠폰" |
| **점검창** | `checkoutTimestamp` ∈ [오늘−8주, 오늘−14일] | ① 미제출 색출. "유예 끝났는데 콘텐츠 없는 건" |

- 점검창 하한(8주)은 무한 누적 방지용 — 8주 넘게 미제출이면 이미 별도 처리 대상으로 간주.
- 두 창은 겹칠 수 있으나 섹션이 분리되어 중복 표기 안 됨.

## 5. 콘텐츠 매칭 (유사매칭) — ⚠️ 정확매칭 폐기

> **결정**: `예약번호` 정확 매칭은 **버린다**. 크리에이터가 `예약번호`를 신뢰성 있게 적지 않기 때문. 대신 **핸들 + 캠핑장 유사이름 + 퇴실일 기준 날짜**로 매칭.

### 5.1 입력
- 쿠폰 예약 1건: `핸들 H`(쿠폰명 괄호 속 = 예: `크리에이터 (포토라이)` → `포토라이`), `캠핑장 C`(=`campName`), `퇴실일 D`(=`checkoutTimestamp`)
- 후보 콘텐츠 행: Airtable 「컨텐츠 업로드」에서 `협찬의 종류 = 캠핑장 예약` & `Created/업로드 날짜` ∈ 점검 lookback(최근 ~10주) 로 사전 필터한 집합

### 5.2 매칭 3축
1. **핸들 축** — `H` ↔ 콘텐츠 `채널명` / `크리에이터 채널명 (from 크리에이터 명단)`
   - 정규화: 공백 제거, 소문자화, 따옴표/특수문자 제거
   - 판정: 정규화 후 동일하거나 한쪽이 다른 쪽 포함
2. **캠핑장 축 (유사매칭)** — `C` ↔ 콘텐츠 `캠핑장명 (from 캠핑장 이름...)`
   - 정규화: ① 공백 전부 제거 ② 지역명 접두/접미 제거(시·도·군·구 + 대표 지역어) ③ 일반 접미어 제거(`캠핑장/오토캠핑장/야영장/글램핑/펜션/캠프/관광농원`)
   - 판정: 정규화 후 한쪽이 다른 쪽을 포함 **또는** 문자 bigram Jaccard ≥ 0.6
   - 예: `여수 작금캠핑장` → `작금` / `작금 캠핑장` → `작금` ⇒ 일치
3. **날짜 축** — 콘텐츠 `업로드 날짜`(없으면 `Created`) ≥ `D − 2일`(퇴실 직전 업로드 허용 버퍼)

### 5.3 판정 결과
| 조건 | 상태 |
|------|------|
| 핸들 ✓ + 캠핑장 ✓ + 날짜 ✓ | ✅ **제출 완료** |
| 핸들 ✓ + 캠핑장 모호(부분일치/임계 미달) | ⚠️ **확인 필요** (사람 검토) |
| 핸들 ✓ 없음 (또는 후보 0) + 퇴실+14 경과 | ❌ **미제출** |
| 예약 status = cancelled | 매칭 제외 → §6.3 취소 섹션으로 |

- ⚠️ "확인 필요"는 미제출로 단정하지 않고 별도 표기 → 오판으로 인한 잘못된 회수 방지.

## 6. 리포트 구성 (camfit-brand HTML)

`anthropic-skills:camfit-design-system` 또는 `camfit-admin-plugin:camfit-brand` 스타일 적용. 단일 HTML 파일.

### 6.1 상단 KPI 카드
- 주간 사용 건수 (활동창)
- **총 협찬비 (할인액 합)** ← 명시 요청, 가장 크게
- 🔴 미제출 N건 (점검창)
- 취소 N건 (활동창)
- 활성 크리에이터 수 (고유 핸들)

### 6.2 🔴 미제출 액션 리스트 (최우선 섹션)
점검창 + ❌/⚠️ 상태. 컬럼: 크리에이터(핸들) · 캠핑장 · 입실\~퇴실 · D+경과일 · 상태(❌/⚠️) · 어드민 링크

### 6.3 취소 모니터
쿠폰 적용 후 취소된 건. 컬럼: 크리에이터 · 캠핑장 · 입실\~퇴실 · 할인액 · 취소시각 · 어드민 링크

### 6.4 이번 주 전체 사용 테이블 (활동창)
컬럼: 사용일 · 크리에이터 · 캠핑장 · 존 · **입실\~퇴실** · 쿠폰명 · 할인액 · 콘텐츠 상태(✅/⚠️/❌/유예중) · 어드민 링크

### 6.5 재사용 크리에이터 랭킹
누적 사용 건수 多 순 (점검 lookback 기준). 핸들 · 사용건수 · 누적 할인액 · 미제출 건수.

### 6.6 캠핑장별 분포
협찬 다발 캠핑장. 캠핑장 · 건수 · 누적 할인액.

> 입실\~퇴실은 전 섹션 공통 표기. 날짜 포맷 `YYYY-MM-DD(요일)`, KST.

## 9. 한계 & 향후

- **유사매칭 오판**: 캠핑장명이 너무 일반적이거나 핸들 표기가 크게 다르면 ⚠️로 빠짐 → 사람 검토 필요. 임계값은 운영하며 튜닝.
- **MCP-in-cron 리스크**: 스케줄 에이전트 헤드리스 실행 시 `probe`/`Airtable` MCP 인증이 끊길 수 있음 → **1차 검증 항목**.
- **향후 B안**: 안정화 후 결정론적 Node 스크립트로 이전 가능 (`MONGODB_URI` 확보 시).
- **예약번호 보강**: 크리에이터 포털 제출 시 `예약번호` 자동 채움이 되면, 유사매칭 → 정확매칭 하이브리드로 정밀도↑.
