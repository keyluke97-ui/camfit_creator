> 작성일: 2026-06-16

## 1. Problem Statement

팔로워 쿠폰 협찬 배포 후 크리에이터에게 **가장 많이 들어오는 문의**:

> "쿠폰 코드를 등록하고 보니, 크리에이터(내 예약) 쿠폰이 아니라 **팔로워 쿠폰 코드를 등록**해서 100% 할인이 안 된다."

한 캠페인에 코드가 2종이다:

| 쿠폰 | 데이터 필드 | 용도 |
|------|------------|------|
| **내 예약 쿠폰** | `couponCode` | 크리에이터가 **본인 캠핏 계정에 등록**해 본인 캠핑 예약에 사용 (협찬 할인) |
| **팔로워 쿠폰** | `followerCouponCode` | 크리에이터가 **팔로워에게 공유** → 팔로워가 예약 시 부분 할인 |

**근본 원인 (UI)**: 신청 완료(ApplicationModal Step 4)에서 두 코드가 **시각적으로 거의 동일**하다 — 둘 다 초록 테두리 박스 + 큰 모노폰트 + 복사 버튼. 그 사이에 "캠핏 쿠폰 등록하러 가기" 버튼이 끼어 있어, 크리에이터가 아래쪽 팔로워 코드를 복사/등록해버린다. "모든 조건 한번에 저장하기" 복사 텍스트에도 두 코드가 나란히 들어가 혼동을 키운다. 입실 등록(CheckinModal)에서도 두 코드가 함께 노출된다.

---

## 2. Goals / Non-Goals

### Goals
1. 신청 완료 직후 **등록 순간**에 크리에이터가 **내 예약 쿠폰만** 명확히 인지·복사하도록 한다.
2. 두 쿠폰의 **역할 구분**(내가 쓸 것 / 팔로워에게 줄 것)을 시각·문구로 분명히 한다.
3. 붙여넣기 실수 자체를 줄인다(등록 버튼이 내 예약 쿠폰을 자동 복사).
4. 입실 등록·복붙 텍스트 등 코드가 함께 노출되는 모든 지점에서 동일한 명확성을 적용한다.

### Non-Goals
- **앱 전체 라이트 테마(DS 2.5) 전환** — 별도 과제로 분리. 이번엔 **다크 유지**.
- 백엔드/API/Airtable 변경 — **없음**. `couponCode`/`followerCouponCode`는 이미 반환됨. **프론트 전용**.
- 파트너 협찬 레거시 경로 — 손대지 않음(CLAUDE.md 지침대로 보존).
- 쿠폰이벤트가 아닌 캠페인(팔로워 쿠폰 없음)의 동작 변경 — 라벨 명확화 외 변경 없음.

---

## 3. Approach (확정: B + C + v2 카피 + Toss 원칙)

**B. 팔로워 쿠폰을 등록 순간에서 분리** + **C. 등록 버튼이 내 예약 쿠폰 자동복사** + **fill/weak 시각 위계** + **"사용하지 마세요(등록은 OK)" 카피**.

### 3.1 핵심 결정
- **fill/weak 2단계 위계** (Toss 2-1/6-3): 내 예약 쿠폰 = fill(초록 강조 hero), 팔로워 쿠폰 = weak(차분한 회색 보조). **경고색(빨강/노랑) 안 씀** (Toss 1-4 "강요 아닌 권유").
- **"등록 금지"가 아니라 "사용 주의"**: 팔로워 쿠폰을 등록해 어떤 쿠폰인지 확인하는 건 괜찮다. **내 예약에 *사용*하지만 않으면 된다.** 부정형 경고 대신 긍정형으로 — "내 예약엔 위 코드를 사용해주세요"(Toss 1-2 사실 전달, 긍정형 라이팅).
- **예측 가능한 CTA** (Toss DP5/P1): "캠핏 쿠폰 등록하러 가기" → **"내 예약 쿠폰 복사하고 캠핏으로 이동"**. 클릭 시 내 예약 쿠폰을 클립보드에 복사하고 **새 탭**으로 등록 페이지를 연다(+ "내 예약 쿠폰을 복사했어요" 피드백). 새 탭 전환은 CLAUDE.md 외부링크 규칙(`target="_blank" rel="noopener noreferrer"`)도 충족(현재 anchor는 미준수).

### 3.2 확정 카피
- 완료 헤드: `신청이 완료되었어요!` (능동/해요체)
- 내 예약 쿠폰 위 안내: `이 쿠폰을 캠핏에 등록하고 예약하면 끝나요`
- 내 예약 쿠폰 라벨: `내 예약 쿠폰 코드` (유지)
- CTA: `📋 내 예약 쿠폰 복사하고 캠핏으로 이동`
- 복사 피드백: `내 예약 쿠폰을 복사했어요`
- 팔로워 쿠폰 헤더: `🎟️ 팔로워 쿠폰 · 내 예약용 아니에요`
- 팔로워 쿠폰 본문: `팔로워에게 공유하는 쿠폰이에요. 등록해서 어떤 쿠폰인지 확인하는 건 괜찮지만, 내 캠핑 예약에는 쓰지 말고 위의 '내 예약 쿠폰'을 사용해주세요.`

## 4. 컴포넌트별 변경 (4파일, 프론트 전용)

### 4.1 `components/ApplicationModal.tsx` (Step 4)
- 내 예약 쿠폰 박스를 **hero**로 강조 + 위에 안내 한 줄 추가.
- 등록 `<a href={COUPON_REGISTER_URL}>` (line ~404) → **onClick 핸들러**로 변경: `couponCode` 클립보드 복사 → `target="_blank" rel="noopener noreferrer"`로 새 탭 오픈 + 복사 피드백 상태. (anchor 유지하되 onClick에서 copy, 또는 button + window.open. anchor+onClick+target_blank 권장 = 보안 규칙 충족.)
- 팔로워 쿠폰 callout(`FollowerCouponCallout`)을 **기본 접힘** + `🎟️ 팔로워에게 공유할 쿠폰 보기 ▼` 토글 안에 둔다(B). 펼치면 4.2의 weak 스타일로 노출.
- "모든 조건 한번에 저장하기" 복사 텍스트(`handleCopyAll`, line ~129): **「내 예약(등록용)」 / 「팔로워에게 공유용」** 두 블록으로 헤더 분리. 미리보기 UI도 동일 분리. (현재 line 446 `• 내 배포 코드:` → `팔로워 쿠폰 코드:`로 통일.)

### 4.2 `components/ApplicationCouponSections.tsx` (`FollowerCouponCallout`)
- 초록(`#01DF82` border/glow) → **weak 회색 톤**(`bg-[#202020] border-[#3a3a3a]`, 코드 텍스트 `#cfcfcf`)으로 변경. 초록 hero와 시각 분리.
- 헤더/본문을 3.2 확정 카피로. 기존 `⚠️ 외부 유출 금지` 한 줄은 유지(보안).
- 단독 callout이 아니라 토글 펼침 컨텐츠로 쓰이도록 props(접힘 상태는 부모 ApplicationModal이 관리).

### 4.3 `components/CheckinSections.tsx` (`CheckinCouponBox`)
- 팔로워 쿠폰 박스에 **`· 내 예약용 아니에요`** 보조 라벨 + 한 줄 안내(`내 예약엔 위 '내 예약 쿠폰'을 사용`)를 weak 톤으로 추가. 초록 border(`border-[#01DF82]/30`) → 회색으로 약화해 위계 분리.

### 4.4 `components/CheckinModal.tsx`
- 내 예약 쿠폰 박스(line ~515) + 등록 버튼(line ~529 anchor)을 ApplicationModal과 **동일 패턴**으로(자동복사 + 새 탭 + 예측 가능 카피).
- "협찬 조건 복사"(`handleCopy...`, line ~232~) 텍스트도 내 예약/팔로워 블록 분리(4.1과 동일 규칙).

## 5. Edge Cases / Non-Regression

- **비쿠폰 캠페인**(`campaign.couponEvent` 없음): 팔로워 영역 자체가 렌더 안 됨 → Step 4는 내 예약 쿠폰 + 새 CTA만. 동작 변화 없음, 카피 명확화만 적용.
- **클립보드 실패**: `navigator.clipboard` 미지원/권한 거부 시 기존 textarea fallback(ApplicationModal `handleCopyAll`에 이미 있음) 패턴 재사용. 복사 실패해도 새 탭 오픈은 진행.
- **자동복사 타이밍**: 새 탭 오픈 전에 `await navigator.clipboard.writeText(couponCode)` 완료 후 `window.open`. 실패해도 사용자는 hero 박스에서 수동 복사 가능.
- **followerCouponCode 빈 값**: 분배 실패/레거시 → 팔로워 토글 자체를 렌더하지 않음(기존 `&& followerCouponCode` 가드 유지).
- **다크 테마 일관성**: 라이트 도입 안 함. weak 톤은 기존 다크 회색 토큰(`#202020`/`#3a3a3a`/`#cfcfcf`) 사용.
